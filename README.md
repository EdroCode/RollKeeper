# RollKeeper

> A dashboard built by roleplayers, for roleplayers.

<img width="300" alt="Logo" src="https://github.com/user-attachments/assets/43d4c82f-a7c0-4473-9f1f-95473a46fabd" />

Running a TTRPG session online means juggling a dozen browser tabs, battle timers, player stats, HP trackers, monster sheets, and more. RollKeeper brings all of those tools into one place, so the Dungeon Master can focus on the game instead of the browser.

## Status

Active development - solo project, early stage.

## Tech Stack

- **[Next.js](https://nextjs.org)**
- **React**
- **TypeScript**
- **Tailwind CSS**

## Getting Started

```bash
npm install   # install dependencies
npm run dev   # start the dev server
```

Then open [localhost:3000](http://localhost:3000).

Other useful scripts:

```bash
npm run build    # production build
npm run lint     # check code with ESLint
npm run format   # format code with Prettier
```

## Got Opinions?

RollKeeper is being built by a roleplayer, for roleplayers, so it should grow with input from the people actually running sessions with it. If there's a feature you're missing, a workflow that feels clunky, or just a strong opinion about how a DM dashboard should work, open an [issue](../../issues) or start a [discussion](../../discussions) and share it.

No idea is too small, what's obvious to one table is a missing piece for another.

## Contributing

This is currently a solo project, but contributions, suggestions, and bug reports are welcome.

1. Fork the repo and create a branch: `git checkout -b feature/your-feature`
2. Make your changes, and run `npm run lint` and `npm run format` before committing
3. Commit with a clear message and open a pull request describing what changed and why

For larger changes, please open an issue first to discuss the idea before investing time in a PR.
